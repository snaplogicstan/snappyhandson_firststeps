SnapLogic Training Guide for Data Analysts

From Tableau Prep to SnapLogic: Hands-On Workshop

🏢 The Business Problem

As data analysts, we regularly receive complaints data from multiple channels and in different formats (CSV, JSON). This data is often messy—Firm Reference Numbers (FRNs) are inconsistently formatted, product categories vary, and missing values are common. Furthermore, to make this data actionable, we need to cross-reference it with our official Salesforce Firm Master database.

This workshop demonstrates how to automate what is traditionally a manual, error-prone data-prep process. We will build an automated data pipeline to unify these disparate sources, standardize formatting, handle missing values, and produce a clean, pivoted Excel report ready for regulatory analysis.

🎯 What We're Building

In this workshop, you'll build a pipeline that:

  1. Combines current complaints (CSV) with historical complaints (JSON)
  2. Enriches data with firm details from Salesforce
  3. Cleans messy data (inconsistent FRN formats, product names, nulls)
  4. Outputs a pivoted summary showing complaints by firm and product type

🛠️ Step-by-Step Instructions

PART A: Create Pipeline and Read Data (Steps 1-11)

Step 1: Create New Pipeline

  1. Click + New Pipeline in your project.
  2. Enter a name like YourName_FirstSteps.
  3. Click Save.

Step 2-3: Open Snap Catalog

  1. Click the + icon in the toolbar to open the Snap catalog.
  2. Search for File Reader.

Step 4-5: Add File Reader for CSV

  1. Drag File Reader onto the canvas.
  2. Double-click to open settings.
  3. In File Path, enter: Q1_2024_Complaints.csv
  4. Click Save.

Step 6: Validate

Click Save in the toolbar. Watch the snap turn green (valid).

Step 7: Add CSV Parser

  1. Search for CSV Parser in the Snap catalog.
  2. Drag it onto the canvas.
  3. Connect the File Reader's output (right side) to CSV Parser's input (left side).
  4. Open CSV Parser - defaults are fine (Header: true).

Step 8: Validate Pipeline

Click the Validate button (checkmark icon) in the toolbar.

Step 9: Repeat for Other Files

Add two more File Reader + Parser chains:

  • For Historical JSON:
    • File Reader1 → File Path: Historical_Complaints.json
    • JSON Parser (not CSV Parser!) → Set Array Elements as Documents: true
  • For Salesforce CSV:
    • File Reader2 → File Path: Salesforce_Firm_Master.csv
    • CSV Parser1

(You should now have 3 File Readers, 2 CSV Parsers, and 1 JSON Parser - all green).

Step 10-11: Preview Data

  1. Click the green circle on the right side of JSON Parser.
  2. You'll see the data as a table.
  3. Notice the lowercase column names: complaint_id, firm_ref, etc. This is our first problem to fix!

PART B: Rename and Combine Data (Steps 12-16)

Step 12-13: Add Rename Columns Mapper

The JSON has lowercase columns (complaint_id) but the CSV has Title Case (Complaint_ID). We need to match them.

  1. Add a Mapper snap after JSON Parser.
  2. Rename it to Rename Columns.
  3. Configure the mapping table:
Expression Target Path
$complaint_idComplaint_ID
$firm_refFirm_Ref
$complaint_dateComplaint_Date
$product_typeProduct_Type
$redress_amountRedress_Amount
$statusStatus
$days_openDays_Open
$complaint_textComplaint_Text

💡 Tip: You can drag fields from the left Input Schema panel and rename them on the right.

Step 14: Add Union

  1. Add a Union snap.
  2. Connect both streams to it:
    • CSV Parser output → Union input0
    • Rename Columns output → Union input1

(The Union stacks both datasets into one).

Step 15: Preview Union - Spot the Duplicates!

Preview the Union output. You'll notice some complaints appear twice (e.g., CMP-000905). We added the same record to both files for training purposes.

Step 16: Add Deduplicate

  1. Add a Deduplicate snap after Union.
  2. Configure:
    • Click the speech bubble icon to get field suggestions.
    • Add $Firm_Ref as the matching field.
    • Comparator: Levenshtein
    • Threshold: 0.8

PART C: Standardise FRN Format (Step 17)

Step 17: Add Mapper1 (FRN Cleanup)

Our FRN data is messy: "FRN-123456", "frn-789012", "123456", "FRN 789012".

  1. Add a Mapper snap after Deduplicate.
  2. Rename it to Mapper1.
  3. Check "Pass through" ☑️ (keeps all other fields).
  4. Add this mapping:
Expression Target Path
'FRN-' + $Firm_Ref.replace(/^FRN-?/i, '').replace(/[-\s]/g, '').toUpperCase() $Firm_Ref

✨ Or use SnapGPT! Click the SnapGPT icon and enter:
"Standardise Firm_Ref to format FRN-XXXXXX. Remove any existing FRN prefix (case insensitive), remove all dashes and spaces, convert to uppercase, then add FRN- prefix at the start"

PART D: Join with Firm Data (Steps 18-21)

Step 18: Add Sort Snaps

For the Join to work efficiently, both streams need to be sorted by the join key.

  • Sort for complaints: Add a Sort snap after Mapper1. Sort by $Firm_Ref (Ascending).
  • Sort1 for Salesforce: Add a Sort snap after CSV Parser1. Rename it to Sort1. Sort by $FRN__c (Ascending).

Step 19-20: Add Join

  1. Add a Join snap.
  2. Connect both sorted streams:
    • Sort output → Join input0
    • Sort1 output → Join input1
  3. Configure:
    • Join Type: Left outer
    • Left Path: $Firm_Ref
    • Right Path: $FRN__c

Step 21: Preview Combined Data

Preview the Join output. You should now see complaint fields AND firm fields (Firm_Name__c, Firm_Type__c, Region__c) together!

PART E: Clean and Enrich Data (Steps 22-23)

Step 22: Add Mapper2 (The Big Cleanup)

This is where we clean everything. Add a Mapper snap after Join.

⚠️ Important: Do NOT check "Pass through" - we want to control exactly what comes out.

Expression Target Path What it does
$Complaint_ID $Complaint_ID Pass through
$Firm_Ref FRN Rename
$Firm_Name__c == null ? 'UNKNOWN FIRM' : $Firm_Name__c.toUpperCase() Firm_Name Handle nulls, uppercase
$Firm_Type__c == null ? 'Unknown' : $Firm_Type__c Firm_Type Handle nulls
$Region__c == null ? 'Unknown' : $Region__c Region Handle nulls
Date.parse($Complaint_Date) Complaint_Date Parse as date
'Q' + Math.ceil(Date.parse($Complaint_Date).getMonth() / 3) + '-' + Date.parse($Complaint_Date).getFullYear() Quarter Create Q1-2024 format
({'mortgage': 'Mortgage', 'insurance': 'Insurance', 'pension': 'Pension', 'investment': 'Investment'}[$Product_Type.toLowerCase()] || 'Other') Product_Type Standardise Title Case
$Redress_Amount == null ? 0 : $Redress_Amount Redress_Amount Null to zero
($Redress_Amount == 0) ? 'No Redress' : (($Redress_Amount < 500) ? 'Under £500' : (($Redress_Amount < 2000) ? 'Under £2000' : '£2000+')) Redress_Band Create bands
$Status Status Pass through
$Days_Open == null ? 0 : $Days_Open Days_Open Null to zero
($Days_Open <= 14) ? 'Fast' : (($Days_Open <= 30) ? 'Standard' : 'Slow') Resolution_Speed Categorise speed

✨ SnapGPT Prompts for complex expressions:

  • Quarter: "Create a Quarter field from Complaint_Date in format Q1-2024"
  • Product Type: "Standardise Product_Type to Title Case: mortgage→Mortgage, insurance→Insurance, pension→Pension, investment→Investment, anything else→Other"
  • Resolution Speed: "Create Resolution_Speed: 14 days or less is 'Fast', 15-30 is 'Standard', over 30 is 'Slow'"

Step 23: Preview Clean Data

Preview Mapper2 output. You should see a beautiful, clean dataset!

PART F: Aggregate and Pivot (Steps 24-28)

Step 24: Add Aggregate

  1. Add an Aggregate snap after Mapper2.
  2. Configure Group By Fields: $Firm_Name and $Product_Type.
  3. Configure Aggregate Fields: Function COUNT on $Firm_Name → Result Field Total_Complaints.

Step 25: Add Sort2

Add a Sort snap after Aggregate. Sort by $Firm_Name (Ascending) and $Product_Type (Ascending).

Step 26: Add Group By Fields

  1. Add a Group By Fields snap.
  2. Configure Fields: $Firm_Name.
  3. Configure Target Field: Firm_Results.
    (This groups all product type rows for each firm into an array).

Step 27: Add Mapper3 (Pivot)

Now we pivot rows into columns - one column per product type. Add a Mapper snap and configure:

Expression Target Path
$groupBy.Firm_Name Firm_Name
$Firm_Results.filter(item => item.Product_Type == 'Mortgage').reduce((sum, item) => sum + item.Total_Complaints, 0) Mortgage
$Firm_Results.filter(item => item.Product_Type == 'Insurance').reduce((sum, item) => sum + item.Total_Complaints, 0) Insurance
$Firm_Results.filter(item => item.Product_Type == 'Investment').reduce((sum, item) => sum + item.Total_Complaints, 0) Investment
$Firm_Results.filter(item => item.Product_Type == 'Pension').reduce((sum, item) => sum + item.Total_Complaints, 0) Pension

✨ SnapGPT Prompt:
"Sum Total_Complaints for all items in Firm_Results where Product_Type equals 'Mortgage'" (Then copy/modify for Insurance, Investment, Pension).

Step 28: Preview Final Output

Preview Mapper3. You should see the pivoted table with firms as rows and product types as columns!

📘 Quick Reference Cards

Expression Reference

String Transformations

  • Uppercase: $Field.toUpperCase()

    SnapGPT: "Convert Field to uppercase"

  • Replace text: $Field.replace('old', 'new')

    SnapGPT: "Replace 'old' with 'new' in Field"

  • Regex replace: $Field.replace(/pattern/g, '')

    SnapGPT: "Remove all dashes and spaces from Field"

Null Handling & Logic

  • Default if null: $Field == null ? 'Default' : $Field

    SnapGPT: "If Field is null, use 'Default'"

  • Nested IF: $A > 100 ? 'High' : ($A > 50 ? 'Med' : 'Low')

    SnapGPT: "Categorise: over 100 High, 51-100 Med, else Low"

  • Lookups: ({'a': 'Apple'}[$Field] || 'Other')

    SnapGPT: "Map 'a' to 'Apple', anything else to 'Other'"

🛠️ Troubleshooting

Problem Solution
Snap is red/yellow Click to open, check error message, verify file paths.
Join returns empty Check both Sort snaps use the correct field names.
Nulls in output Add null handling in Mapper: $Field == null ? 'Default' : $Field.
Wrong column names Check Rename Columns mapper has correct mappings.
Pivot shows zeros Verify Product_Type spelling matches exactly (case-sensitive).
